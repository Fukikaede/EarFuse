# EarFuse Menubar（MVP スキャフォールド）

これは、リアルタイムのレベル監視と聴覚保護用「ヒューズ」緊急保護のための macOS メニューバーアプリのスキャフォールドです。

## 現在の実装範囲（Milestone 1 基盤）

- メニューバーアプリ + リアルタイム `Peak/RMS` 表示。
- `Backend A`：既定でシステム出力キャプチャ（Core Audio 出力デバイスコールバック）を優先。
- システム出力が利用できない場合は、入力デバイス（マイク / オーディオインターフェース）へ自動フォールバック。
- 安全ステートマシン（`safe/yellow/red`）を実装し、「継続時間 + ヒステリシス」戦略を採用。
- メニューバーポップオーバー内で `Production/Listening` をワンタップ切り替え。
- メニューバーポップオーバーで `Capture Source`（System Output / Input Device / Mock）を切り替え可能。
- 直近 60 秒の履歴グラフ（RMS メイン線 + Peak 細線）としきい値カラーバンド。
- ヒューズエンジンの骨組み（システム音量制御インターフェースを抽象化済み）。
- イベントログをローカル JSON に永続化し、今日 / 今週の危険時間と最新イベントを表示。
- `Meter/Policy` のユニットテスト用サンプルファイルを事前配置。

## ビルドと実行

```bash
cd /Users/kaede/Codex/earfuse-menubar
swift build
swift run EarFuseApp
```

補足：現在の環境には Command Line Tools のみが入っています。macOS アプリの完全なパッケージング、署名、権限設定を行う場合は、フル版 Xcode が入ったマシンでこのプロジェクトを開いてください。
初回起動時にマイク権限を要求します。未許可の場合、リアルタイムレベルは更新されません。
システム出力キャプチャが環境制約で使えない場合は、入力デバイスへ自動フォールバックします。

## リポジトリ構成

- `App/`：アプリのエントリ
- `MenuBarUI/`：メニューバー Scene とポップオーバー UI
- `SettingsUI/`：設定関連の共通 UI コンポーネント
- `Audio/`：収集、計測、監視サービスのオーケストレーション
- `Core/`：ポリシー、設定、アラート、ヒューズ、ログ、共有モデル
- `Tests/UnitTests/`：ユニットテストサンプル（完全なテスト環境統合は未対応）
- `docs/TechnicalDesign.md`：技術設計ドキュメント
